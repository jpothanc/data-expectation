# Log Files

This directory contains daily log files generated by the generator.

## Log File Format

- **File naming**: `generator_YYYY-MM-DD.log` (e.g., `generator_2026-01-03.log`)
- **Rotation**: Log files rotate daily at midnight
- **Retention**: Log files are kept for 30 days, then automatically deleted
- **Encoding**: UTF-8

## Configuration

The log directory path can be configured in `config.json`:

```json
{
  "logging": {
    "log_dir": "log"
  }
}
```

You can use:
- Relative paths: `"log"`, `"logs/generator"`, `"../logs"`
- Absolute paths: `"C:/Projects/logs/generator"`, `"/var/log/generator"`

## Log Levels

- **INFO** (default): General information about validation runs
- **DEBUG** (with `--verbose` flag): Detailed debugging information including API calls and database operations

## Log Format

Each log entry includes:
- Timestamp: `YYYY-MM-DD HH:MM:SS`
- Logger name: Module that generated the log
- Level: INFO, DEBUG, WARNING, ERROR
- Message: Log message content

Example:
```
2026-01-03 14:30:15 - generator.src.core.validator - INFO - Starting validation for apac/stock/XHKG
2026-01-03 14:30:16 - generator.src.api.api_client - DEBUG - Calling API: http://127.0.0.1:5006/api/v1/rules/validate/stock/XHKG
```
